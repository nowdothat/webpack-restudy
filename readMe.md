多入口模式，切记不能使用 html-webpack-plugin 的 inject 功能
问题来源：
假如我一个项目里，允许有A、B两个html文件，他们之间是不同的入口文件（比如一个是用户入口页，一个是管理入口页）。

显然虽然是两个不同的入口，但是他们之间有很多共通的逻辑（否则就有大量重复开发工作了），因此我们需要将其写在同一个工程中，然后通过不同的入口文件引入他。

他的依赖树可能是这样的：

```
.
|____first.html
| |____first.js
| | |____common.js
|____second.html
| |____second.js
| | |____common.js
```
通过这样的打包模式-就将 common.js 的所有内容都打包到了 first, second 中, 而各自的私有方法各自保留
